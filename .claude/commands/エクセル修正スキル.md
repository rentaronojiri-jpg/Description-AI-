# エクセル修正スキル

openpyxl で生成した Excel ファイルの罫線・文字切れ問題を診断・修正するスキル。

## このスキルが解決する問題

- セル結合（merge_cells）を使うと**先頭セル以外に罫線が設定されず、横線が途中で途切れる**
- 列幅・行高さが不足して**文字が切れる**
- 余分な列（I列など）にボーダーなし設定が入り込み**右端の線が消える**

## 実行手順

以下の手順で Python スクリプト（openpyxl）の罫線問題を修正する。

### 1. スクリプトを読み込んで問題箇所を特定

- `merge_cells()` を呼ぶ前に先頭セルにしか `border` を設定していない箇所を探す
- `Border()` （引数なし＝罫線削除）を設定している列を探す
- 列幅・行高さが日本語文字数に対して不足していないか確認する

### 2. 修正の鉄則（必ず守る）

```python
# NG: merge 後に先頭セルだけ罫線を設定 → 途中で線が切れる
ws.merge_cells(start_row=r, start_column=1, end_row=r, end_column=8)
ws.cell(r, 1).border = b_thin()   # セル2〜8 に罫線なし！

# OK: merge 前に全列へ罫線を引いてからマージする
def stamp(ws, row, c1=1, c2=8):
    for col in range(c1, c2 + 1):
        ws.cell(row, col).border = b_thin()

stamp(ws, r)           # ← 先に全列へ細線を適用
ws.merge_cells(...)    # ← その後マージ
ws.cell(r, 1).value = "セクション名"  # 値・スタイルは最後
```

### 3. その他の修正ポイント

- **余白・スペーサー列**：`Border()` で罫線を消すのではなく、白背景（`"FFFFFF"`）で視覚的に空けて罫線は維持する
- **列幅**：日本語 10 文字 ≈ 幅 20 を目安。長い文字列がある列は +5〜8 余裕を持たせる
- **行高さ**：折り返し（wrap=True）を使う行は最低 36pt 以上にする

### 4. 修正後の確認

1. スクリプトを再実行して xlsx を生成
2. Excel で開き「全セルを選択 → ホーム → 罫線」で均一性を確認
3. 印刷プレビューで文字切れがないことを確認

## 使用例

```
/エクセル修正スキル
→ 現在のディレクトリにある generate_excel_template.py を読み込み、
   上記の手順で罫線・文字切れを修正して再生成する
```
